# Анатомия иконок — справка

Статика гарнитуры описывается декларативно: `semantics/grid.json` (токены
системы) + `semantics/anatomy.json` (декларации глифов). Файлы SVG со
статусом `generated` — производные от деклараций; гейт дрейфа держит
совпадение ≥ 99.5% чернил. Правка таких иконок = правка декларации +
перегенерация, не ручное редактирование path.

## Конвенция единиц

| Величина | Единица |
|---|---|
| Длины, координаты, радиусы | **доли канвы** (0…1); резолв ×`canvas.width` в одной точке (`buildGlyph`) |
| Углы | градусы |
| Счётчики (зубья, спицы) | штуки |
| Коэффициенты (ζ, показатель n) | безразмерные |

Смена размера канвы масштабирует всю дисциплину пропорционально.

## Токены системы (grid.json)

| Токен | Номинал (канва 24) | Смысл |
|---|---|---|
| `strokeWidth.base` | 1.8 | Regular — штрих глифа на канве |
| `strokeWidth.bold` | 2.4 | Bold — filled-начертание нефиллебельных штриховых (checkmark, reload, стрелки) |
| `strokeWidth.containerGlyph` | 2.0 | глиф внутри контейнера (компенсация уменьшенного габарита) |
| `strokeWidth.enclosureRing` | 1.5 | кольцо-обрамление (контейнер оптически легче глифа) |
| `strokeWidth.capRadius` | 0.9 | радиус капа |
| `cornerSmoothing` (ζ) | 0.6 | профиль скруглений Figma: радиус вершины неизменен, сглаживается только вход в дугу; применяется ТОЛЬКО к задекларированным скруглениям прямых углов (не пост-обработка; стык с дугой = G1-fillet) |

Веса — независимые абсолютные базисы (как Regular/Bold шрифта), не формула.

## Архетипы глифов (anatomy.json)

| Архетип | Пример | Суть |
|---|---|---|
| `radial-gear` | cog | венец из N зубьев × токены, вырезы с G1-fillet |
| `arc-terminal` | reload | дуга-скелет × вес + терминал-актив (кривыми), стык by construction |
| `stroke-v` | chevron-down | чернильные якоря концов инвариантны весу, вес растёт наружу |
| `container-glyph` | chevron-down-circle | кольцо/диск из токенов + глиф; filled = диск − глиф |
| `rounded-rect-container` | tablet-* | рамка: внешний контур (ζ) + внутренний офсет пером |
| `composite` | pause, time, component | список частей-примитивов с режимом на вариант |

Примитивы composite: `rounded-rect` (+rotation, бюджет Figma),
`circle-dot`, `superellipse`, `superellipse-stroke` (офсеты оси по
нормали — перо константно), `clock-hands`, `rounded-rect-cutout`,
arc-band (волны). Кривые сериализуются кубиками (адаптивная выборка по
повороту касательной); полигонализация запрещена.

## Законы пары Outline ↔ Filled

- Силуэт filled одноконтейнерных = внешний контур outline.
- Штриховые нефиллебельные: filled = Bold-начертание (1.8 → 2.4), осевые
  длины инвариантны весу.
- Внешний чернильный габарит глифа инвариантен варианту; компенсация — внутрь
  (столбики pause сжимаются к наружным краям, точки ellipsis растут и стягиваются).
- Негативы (вырезы) могут быть толще штриха (стрелки time: 2.0 при штрихе 1.8).
- Регистрация пары: канон 0.15 (`check-variant-parity`).

## Гейты (все в `pnpm verify` и CI)

| Гейт | Что держит |
|---|---|
| `check:anatomy` | дрейф файлов от деклараций (generated ≥ 99.5% hard, hand ≥ 95% report) |
| `check:variant-parity` | контракт пары O↔F: каноны колец, регистрация |
| `check:path-quality` | шум кривых: изломы, лишние узлы, волосяные фрагменты (класс дырок), встык-швы между path |
| `check:static-grid` | поля и keylines |

## Как перегенерировать иконку из декларации

```js
import { buildGlyph } from './scripts/lib/anatomy-gen.js';
const grid = JSON.parse(readFileSync('semantics/grid.json', 'utf8'));
const anatomy = JSON.parse(readFileSync('semantics/anatomy.json', 'utf8'));
const { outline, filled } = buildGlyph(anatomy.glyphs['component'], grid);
// outline/filled — готовые d-строки; файл пишется с fill-rule по конструкции:
// evenodd для вложенных контуров, nonzero при нахлёстах частей
```

После записи файлов обязательны `pnpm check:anatomy` (дрейф) и `pnpm verify`.
