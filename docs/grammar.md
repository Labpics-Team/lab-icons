# Грамматика начертания — систематизация без декомпозиции

## Проблема

63 из 222 иконок покрыты генеративными декларациями в
`semantics/anatomy.json` — архетипами и композициями примитивов (круг,
скруглённый прямоугольник, капсула). Остальные **159 — кастомные силуэты**:
swoosh стрелки, гора в «image», крышка «archive», лента «bookmark». В круг и
прямоугольник они не раскладываются.

## Решение: грамматика, а не декомпозиция

Кастомные силуэты систематизируются не тем, **из чего** собраны, а **по
каким правилам построены** — как шрифтовая foundry систематизирует буквы
(они «тоже все разные»): единое перо, единые терминалы, единые суставы,
единый overshoot накладываются на произвольный скелет. `wallet` и `trophy`
не сводятся к кругам, но обе рисуются одним пером, с одинаковыми капами,
ζ-суставами и клиренсом.

Это же принцип SF Symbols: символы не декомпозируются в примитивы —
они держат общую грамматику (весовая ось = шрифт San Francisco,
консистентные терминалы/суставы, оптическое выравнивание, роли слоёв).

## Словарь рёбер мал

Замер корпуса (11 382 сегмента Outline): линия 35%, дуга 24%, кубика 21%,
квадратичная 10%. Любой силуэт — последовательность из четырёх типов
сегментов. Грамматика накладывается **квантованием параметров сегмента**
(не формы): направление → шкала углов, радиус → конструкция, вес → лестница,
поворот → ζ. Это и есть «систематизация безье» — на уровне сегмента.

## Законы грамматики (токены `grid.json`)

| Закон | Токен | Гейт |
|---|---|---|
| Направление прямого ребра | `angleScale` [0,30,45,90,135] + `grammarSnapDeg` 4 | `check-grammar` (v1) |
| Вес штриха | `strokeWidth` лестница 1.5/1.8/2.0/2.4 | `check-variant-parity` (кольца) + `check-ink-weight` (фактическая толщина любого штриха; кламп оси — `--axes`) |
| Угол/сустав | `cornerSmoothing` ζ=0.6 | генераторы; фит `corner-fitter` |
| Терминал | `capRadius` 0.9 | генераторы |
| Охранный зазор | `clearanceMin` 0.8 | `check-static-grid` |
| Оптический центр | `opticalCenterBias` 0.5 | (per-icon, роли-исключения) |
| Overshoot круга | keylines: круг 22 / квадрат 20 (+10%) | `check-static-grid` |

**v1 (реализован):** закон направлений рёбер. Прямое ребро ≥1.0 с наклоном
в полосе (0.5°, 4°) от шкалы обязано снапнуться (снос 1–3° = экспортный
дребезг, видимая кривизна прямой; дальше 4° = намеренная диагональ). Гейт
report-режимом даёт punch-list миграции.

**Реализовано дальше:** закон веса — `check-ink-weight` меряет фактическую
толщину каждого штриха слепо по чернилам (report-каталог + HARD для
промоутнутых глифов из `semantics/ink-weight-promoted.json`).
**Расширяется:** закон терминала (каждый открытый конец = cap).

## Два слоя анатомии

| Слой | Артефакт | Покрытие | Роль |
|---|---|---|---|
| Скелет | `dist/anatomy.json` (`build-anatomy.js`) | все 444 | дескриптивная геометрия: контуры, центроиды, вырезы, симметрия |
| Декларации | `semantics/anatomy.json` (`anatomy-gen.js`) | архетип-выполнимые | генеративная истина: генерат + гарантия дрейфа |
| **Грамматика** | `grid.json` токены + `check-grammar` | **все силуэты** | конструктивные законы поверх любого скелета |

Скелет — что нарисовано. Декларация — как построить (там, где сводится к
архетипу). Грамматика — по каким правилам, для любой формы.

## Перспектива: свой шрифт

Грамматика делает шрифт дорожной картой, а не фантазией: иконка и буква —
один объект под разными именами (общее перо, терминалы, суставы, overshoot,
оптика на разном скелете). Лестница весов → весовая ось шрифта; ζ-сустав →
поворот штриха; cap → терминал буквы; overshoot +10% → «O» перерастает «H».
Стиль-цель — неогротеск класса Inter / Geist / Golos / Anthropic Sans:
рациональная инженерная конструкция, ровно тот характер, что задан иконкам.
Полный текстовый шрифт — отдельный эпик (300+ глифов × веса × кернинг ×
хинтинг); реалистичный первый шаг — вариативный icon-font из самих пиктограмм.
