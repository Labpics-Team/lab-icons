# Анатомическая модель иконок — ADR

Статус: принят. Этап 1 реализован 2026-07-03; части (этап 2) размечены
декларативно — `semantics/layers.json` (per-icon слои, гейт `check:layers`),
поверх них живёт рантайм анимаций `./animate` (`src/animate/index.ts`).
Владелец видения: Даниил.

## Видение

Иконка labpics — не картинка, а **машинно-читаемая анатомия**: знак состоит
из именованных частей с геометрией и смыслом, поверх которых строятся
анимации класса SF Symbols — **по смыслу жеста, не по трансформации кадра**:
песочные часы переворачиваются, кисть красит, колокол качается за язычок.
Signifier — не рисунок, а роль знака; рисунок — одна из проекций анатомии.
«Тиски» стиля держат вариативность: гарды пайплайна (краска, id, парность) —
первые тиски; анатомические инварианты — вторые; вариативные штрихи и
суперэллипсные конструкции — следующие степени свободы внутри тисков.

## Слои модели

1. **Скелет** (этап 1, `dist/anatomy.json`) — детерминированная геометрия,
   добытая из path-данных: контуры (замкнутость, площадь со знаком, вырезы
   по противо-ориентации, bbox, центроид, периметр), агрегаты знака
   (bbox/площадь/центроид), зеркальные overlap-метрики симметрии.
   Ноль придуманных значений: каждая цифра воспроизводима из SVG
   (`scripts/build-anatomy.js`, кривые сэмплируются N=16 — детерминизм).
2. **Семантические привязки** (`anatomy/bindings.json`) — DECLARED-слой:
   жест (`flip`, `sweep`, …), ось, pivot ИЗ скелета (centroid | bbox-center |
   обоснованная точка). Каждая привязка несёт `$why` — обоснование выбора
   оси из физики жеста; голая привязка без обоснования не проходит ревью.
3. **Части** (этап 2) — распознавание именованных частей знака: круговой
   корпус (циферблат), стрелки, язычок колокола — кластеризацией контуров
   скелета + декларативной разметкой спорных. Даёт точные pivot'ы
   (центр окружности вместо центроида знака) и per-part анимацию.
4. **Конструктив** (этап 3) — вариативные штрихи и суперэллипсные примитивы:
   анатомия становится ИСТОЧНИКОМ рисунка (машинное построение), SVG —
   проекцией. Совместимость: соответствие построенного знака текущему SVG
   доказывается той же скелет-метрикой (Хаусдорф контуров — инструментарий
   corner-K labui переиспользуется).

## Гард (`scripts/check-anatomy.js`)

Полнота 444, конечность каждой метрики, домен (bbox ⊂ viewBox±0.5,
площадь > 0), **байт-детерминизм** артефакта (пересборка = файл; ручные
правки и недетерминизм кусаются), валидность привязок (имя существует,
pivot-режим известен модели). Входит в `pnpm verify`.

## Что этап 1 сознательно НЕ делает

- Не рендерит even-odd/nonzero — фиксирует вырезы топологически
  (противо-ориентация внутри path-элемента).
- Не распознаёт части — центроиды знака целиком (честная точность этапа
  задекларирована в `$why` привязок, где это ощутимо: timer).
- Не анимирует — привязки суть данные для движка анимации
  (потребитель — lab-motion; этап впоследствии состоялся: прекомпилированные
  хореографии из `@labpics/motion/presets` + WAAPI-рантайм `./animate`).
